1 Getting Started {#gettingStarted}
2 =========================
3 
4 
5 I. Create new Maven project
6 ---------------------------
7 
8 1. **Define `pom.xml` file**
9 > An example is [here](../../docs/examples/pom.xml).<br>
10 > <i><b>Note</b>: `'InsertValueHere'` text should be replaced with company or project specific values.</i>
11 <br>
12 
13 2. **Define Spring context**
14 > An example is [here](../../docs/examples/ApplicationContext.xml).<br>
15 > <i><b>Note</b>: `'InsertValueHere'` text should be replaced with company or project specific values.</i>
16 <br>
17 
18 3. **Create property files in `resources` directory**
19 > Examples:<br>
20 > – [store.properties](../../docs/examples/store.properties) (store main URL, HTTP and HTTPS ports)<br>
21 > – [pages.properties](../../docs/examples/pages.properties) (store page URLs and titles)<br>
22 > – [jbehave.properties](../../docs/examples/browser.properties) (browser name, number of threads, metafilters and mandatory jbehave settings)<br>
23 > – [browser.properties](../../docs/examples/browser.properties) (proxy & loading timeout settings)<br>
24 > – [log4j.properties](../../docs/examples/log4j.properties) (logging level & main log patterns)<br>
25 > – [mobile.properties](../../docs/examples/mobile.properties) (mobile apps data)
26 <br>
27 
28 4. **Develop tests using BDD approach**
29 > a) Write test stories <i>(in Given-When-Then style, find detailed syntax & examples [here](http://jbehave.org/reference/stable/story-syntax.html)).</i><br>
30 > b) Describe pages <i>(in new classes that extend [AbstractPage](@ref com.griddynamics.qa.ui.AbstractPage), then add these classes to Spring context, add URL & titles to <code>pages.properties</code> file).</i><br>
31 > c) Develop implementation of project-specific steps <i>(common steps should be re-used from [CommonPageSteps](@ref com.griddynamics.qa.ui.steps.CommonPageSteps) class).</i>

35 II. Run tests
36 -------------
37 
38 ### • Desktop browser
39 
40 Common tests execution parameters are as follows:
41 <pre><code>-Dthreads=1 -Dsuite.all=**/*Suite.java -Dsuite.list=SmokeWebSuite clean test -P runTests</code></pre>
42 
43 > Where<br>
44 > <b>–Dthreads</b> : Number of threads used to execute tests<br>
45 > <b>–Dsuite.all</b> : template for all suites to be included in execution <i>(empty value is not allowed; * wildcard can be used)</i><br>
46 > <b>–Dsuite.list</b> : specific suite(s) from template that should be executed <i>(can be empty or not specified – then all suites matching `suite.all` will be executed)</i><br>
47 > <b>–Dstory.list</b> : specific stories belonging the suites that should be executed <i>(can be empty or not specified – then all stories in suite(s) will be executed)</i><br>
48 <br>
49 
50 
51 ### • Mobile browser & mobile apps
52 
53 > <i><b>Note</b>: Please make sure you've installed [mobile prerequisites](@ref prerequisites-mobile) first.</i>
54 
55 Common tests execution parameters are as follows:<br>
56 <pre><code>-Dmobile=android_app -Ddevice=true -Dsuite.all=**/*Suite.java -Dsuite.list=AndroidAppSuite clean test -P runTests</code></pre>
57 
58 > Where<br>
59 > <b>–Dmobile</b> : mobile platform and type of application to test.<br>
60 > *Possible values: `android_app`, `android_browser`, `ios_app`, `ios_safari`.*<br>
61 > <b>–Ddevice</b> : if true real device is used, if false or not specified an emulator is used.<br>
62 > <b>–Dsuite.all</b>, <b>–Dsuite.list</b>, <b>–Dstory.list</b> : see above.
